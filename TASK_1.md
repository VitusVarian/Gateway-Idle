Task: Create an Idle/Clicker Game Planning Workbook

Objective:
Research the "idle" / "clicker" game genre and produce a structured planning document that a user can complete. Once filled in, this document must function as a practical blueprint for future game implementation.

Output location and file:
- Save the document in the `Task Output` folder.
- Filename: `TASK_1_OUTPUT_Idle_Clicker_Planning_Workbook.md`

Primary deliverable requirements:
1. Genre research summary
	- Summarize key characteristics of idle/clicker games.
	- Cover common player loops, progression systems, and monetization patterns.
	- Identify what makes these games engaging over long sessions.
2. Comparative design insights
	- Include at least 5 notable design patterns used in successful idle/clicker titles.
	- For each pattern, provide:
	  - What it is
	  - Why it works
	  - Common implementation pitfalls
3. Planning questionnaire for implementation
	- Provide clear, answerable questions grouped by topic.
	- Questions should be specific enough to inform development decisions.
	- Include short guidance notes where needed.
4. Design considerations checklist
	- Include practical considerations for balance, pacing, retention, UI clarity, and technical feasibility.
	- Add a checkbox-style checklist the user can mark while planning.
5. Personalization and desirable elements section
	- Include a dedicated section where the user can state preferred themes, mechanics, aesthetic style, tone, accessibility needs, and platform priorities.
	- Provide prompts for both "must-have" and "nice-to-have" features.

Required document structure (use these exact section headings):
1. `Purpose and How to Use This Workbook`
2. `Idle/Clicker Genre Research Summary`
3. `Core Design Patterns (with Pros, Cons, and Pitfalls)`
4. `Implementation Planning Questions`
5. `Design Considerations Checklist`
6. `Personalization and Desired Elements`
7. `Completed Plan Summary (To Fill After Answering)`

Quality standards:
- Use concise, practical language (avoid vague theory).
- Prioritize actionable planning guidance over general commentary.
- Ensure questions can directly translate into feature, system, or technical decisions.
- Keep formatting clean and scannable with headings and bullet points.

Acceptance criteria:
- The output file exists in `Task Output` with the required filename.
- All 7 required sections are present with meaningful content.
- The document includes both research insights and user-fillable planning prompts.
- A user who answers the prompts can hand the result to a developer as a credible implementation plan.
